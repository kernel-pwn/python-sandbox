# PythonProject — my Python learning notebook

This repository is my personal place to collect Python things I learn: short examples, exercises, and small projects I build while practicing. It's intentionally informal — code, notes, and experiments live here so I can revisit them later.

## How I use this repo
- Keep small, focused scripts that demonstrate one concept (variables, loops, functions, collections).
- Add short exercises that match the examples so I can practice.
- Build tiny projects (games, utilities) to combine concepts and try new ideas.
- Edit files freely — this is a workspace for exploration, not production.

## Quick overview
- Language: Python (plain .py files)
- Required Python: >= 3.14 (declared in pyproject.toml)
- Dependencies: none (no third-party packages required)

Repository layout
```
basics/         # short example scripts (syntax, control flow, functions)
exercises/      # practice exercises corresponding to basics
mini_projects/  # small runnable programs: games, tools, demos
pyproject.toml  # minimal project metadata (no dependencies)
uv.lock         # lockfile
.idea/          # optional IDE metadata (can be ignored)
```

Selected files
- basics/main.py — example entry-point pattern (if __name__ == "__main__")
- mini_projects/banking_program.py — interactive CLI demo
- mini_projects/slot_machine.py, quiz game.py, dice roller program.py — practice apps

## Notes / personal rules
- Keep examples short and well-commented so I can read them later.
- When I learn a new concept, I add one example + one small exercise.
- Prefer descriptive filenames (consider renaming files that contain spaces).
